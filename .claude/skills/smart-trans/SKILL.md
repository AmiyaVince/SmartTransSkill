# Smart Translation

## When to Trigger

触发条件（满足任一即触发）：
- 用户输入单个英文单词或英文短语，要求翻译、解释、辨析
- 用户消息中包含"翻译"、"什么意思"、"怎么用"、"辨析"等关键词并指向英文词汇
- 用户输入纯英文单词/短语，没有其他中文指令（视为查询请求）
- 用户输入"统计"、"历史统计"、"复习统计"、"查询统计"、"我的词汇量"等 → 触发统计模式

不触发条件（跳过此技能）：
- 用户输入的是完整英文句子（非短语），目的是翻译整句而非学习词汇 → 建议用户明确是否要拆解其中词汇
- 用户输入中文词汇询问英文怎么说 → 这是反向翻译，告知用户本技能专注于英→中解析
- 用户输入非英语内容（法语、日语等）→ 告知暂不支持该语种

## Workflow

收到查询后，按以下流程执行：

### Step 1: 查询在线词典 API

使用 `Bash` 工具执行 `curl` 查询 Free Dictionary API，并通过 `python3 -m json.tool` 格式化：

```bash
curl -s "https://api.dictionaryapi.dev/api/v2/entries/en/{word}" | python3 -m json.tool
```

从返回的 JSON 数组（第一个元素 `[0]`）中提取：
- `phonetic` / `phonetics[].text` — 音标
- `meanings[].partOfSpeech` — 词性
- `meanings[].definitions[].definition` — 英文释义
- `meanings[].definitions[].example` — 例句（该 API 常为空，需模型补充）
- `meanings[].synonyms[]` — 同义词（该 API 常不完整，需模型补充）
- `meanings[].antonyms[]` — 反义词（该 API 常为空，需模型补充）

**错误处理**：
- API 返回 `"title": "No Definitions Found"` 且 HTTP 404 → 检查拼写并提示用户
- `curl` 连接超时或 DNS 解析失败 → API 不可用，回退使用模型训练知识，并在回答开头标注 `> ⚠️ 在线词典暂不可用，以下为模型知识库内容，仅供参考`
- API 返回了数据但某些字段为空 → 正常，这些字段由 Step 2 模型知识补充

### Step 2: 补充模型知识

API 数据可能不完整。对 API 未覆盖的维度，用模型知识补充：
- 中文翻译（API 不提供中文，必须由模型生成）
- 形近词（API 不提供，由模型知识补充）
- 同义词/反义词的细微辨析（API 只列词，辨析由模型补充）

**在补充知识前**，查阅以下参考文件以统一标准：
- 词性标注时参考 `/Users/huangjiadi/Downloads/HuangHa0Ran/英语学习/SmartTransSkill/references/pos-tags.md`——使用其中定义的标记规范（如区分 `vt.` / `vi.`、`[C]` / `[U]`、语域标注等）
- 如果单词包含明显的词根/词缀结构（如 anti-、pre-、-able、-tion），查阅 `/Users/huangjiadi/Downloads/HuangHa0Ran/英语学习/SmartTransSkill/references/roots-affixes.md`，并在回答中简要标注构词分解，帮助用户举一反三

### Step 3: 按格式输出

按下方 Output Format 组织内容。遵循必选/可选规则。

### Step 4: 记录查询历史

```
### 🔄 同义词
- **{单词1}**：{含义} — 辨析：{与原词的细微差别}
- **{单词2}**：{含义} — 辨析：{与原词的细微差别}

- **{单词1}**：{含义}
- **{单词2}**：{含义}
```

### 板块规则

| 板块 | 必选/可选 | 说明 |
|------|----------|------|
| 词性与释义 | **必选** | 至少包含一个词性的完整释义 |
| 例句 | **必选** | 至少 2 条，优先使用 API 提供的真实例句 |
| 形近词 | 可选 | 无合适内容时注明"无常见形近词" |
| 同义词 | 可选 | 无合适内容时注明"无常见同义词" |
| 反义词 | 可选 | 无合适内容时注明"无常见反义词" |

## Anti-Hallucination Rules

1. **API 优先**：英文释义和例句优先使用 API 返回值；仅在 API 无数据时使用模型知识
2. **标注来源**：API 提供的例句后缀标注 `[词典]`；模型生成的例句后缀标注 `[AI]`
3. **不确定时标注**：对不熟悉的生僻词或专业术语，在释义前加 `> ℹ️ 该词较为生僻/专业，以下解析仅供参考`
4. **不编造**：形近词、同义词、反义词如无把握，宁可留空标注"无常见XX"也不硬凑
5. **中文翻译精准性**：每个义项的中文翻译应给出最贴切的 1-2 个对应词，避免罗列过多干扰阅读

## Multi-Word Query Handling

- **短语/固定搭配**（如 "give up", "take off"）：按短语整体解析，词性标注为"短语 (phrasal verb / idiom 等)"，不拆解单词
- **多个单词辨析**（如 "affect vs effect"）：分别解析每个单词，最后增加一个"### ⚖️ 对比总结"板块，表格列出关键区别
- **含数字/符号的输入**：先 strip 掉首尾空白和标点，提取核心英文部分

## 统计模式 (Stats Mode)

当用户要求查看统计时，执行以下步骤：

1. **读取历史文件**：读取 `/Users/huangjiadi/Downloads/HuangHa0Ran/英语学习/SmartTransSkill/history/query-log.md`
2. **解析记录**：提取所有查询记录（排除表头行），统计以下维度：

   - 📊 **总查询数**：历史上共查过多少个单词
   - 📅 **今日查询数**：当天查了几个词
   - 🔥 **最近 7 天查询数**：近一周活跃度
   - 🏆 **最近查询的 5 个词**：日期 + 单词 + 中文意思

3. **按以下格式输出**：

```

| 1 | ... | ... | ... |
| 2 | ... | ... | ... |

| 4 | ... | ... | ... |
```
